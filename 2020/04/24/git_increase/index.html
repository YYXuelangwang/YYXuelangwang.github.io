git increase

发表于 2020-04-24

写这篇文章的目的是为了缓存自己在git上的提示/升级路程，在平时使用过程的记录等；在文章不断增长的过程中，会把部分内容提取出来单独作为一个新的篇章记录说明；

git stash

缓存处处理，当自己有调试代码穿插在项目中时，而这时由于自己或项目其他的人的观念不一致，不允许自己将调试代码上传到仓库时；当当当，你就可以使用这个来管理了，so easy

那上面就是他的一些用法了，我就拿几个叨叨

  1. 等同于git stash save创建缓存，message是你缓存的说明；

  2. 查看缓存列表

  3. 应用缓存后将缓存删除，有点类似栈的操作，num代表你要操作的缓存序列号

  4. 删除缓存

  5. 应用缓存，但是不会删除缓存

git tag

给自己的分支打tag/标签，

  1. 罗列出来已经创建的tag

  2. 创建tag，xxx代表tag的值，版本号啊啥的，message代表说明

  3. 推送指定的tag到远程仓库

  4. 删除指定的tag分支

  5. 删除远程仓库的标签/tag
